Welcome to the English to Bash study!

Thank you for participating in this study! We evaluate whether a tool can help programmers to write Bash scripts.

In the experiment, you will be presented with 16 short file system tasks. For example, "Show the number of lines in file foo.txt". Your objective is to solve each task with a Bash one-liner: a sequence of bash commands (a mini bash script) that is contained on one line. For example, cat foo.txt | wc -l.

You may use any tool to help you solve the tasks; for example, you might read man pages or search on Google.

Instructions

Prerequisites:

  1. Boot up a CSE Linux Home VM.
  2. Install Meld in the VM.
  3. Run $ meld in your VM's terminal. If it opens a GUI window, then Meld is installed; close the window. Otherwise, run $ sudo yum install meld and then double-check that Meld is installed.

Experiment:

  1. Open a new terminal (command shell) and pasting the following into it: 
              cd
          mkdir en2bash
          cd en2bash
          wget https://homes.cs.washington.edu/~tschweiz/research/en2bash-study/experiment.zip
          unzip experiment.zip
          bash
          source experiment/configure
  2. Follow the prompts in your terminal. There are 2 training questions to familiarize yourself with the tool and then the experiment tasks.

Survey:

  1. When you have finished the experiment, please complete this short survey (5 minutes).

Clean-up: (optional)

  1. Remove the experiment files from your Home VM:
    rm -rf ~/en2bash
